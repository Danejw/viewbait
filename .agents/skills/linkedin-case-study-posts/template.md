# LinkedIn case study post template

Copy, fill, delete bracket placeholders, then run human voice + style passes.

```
[Title: what you learned or what you built — outcome first]

**Project:** [Product name]
**Link:** [https://example.com]

**The app**
[2–4 sentences. What the product is. Who uses it. What this specific system/feature does in plain language.]

**The problem**
[2–4 sentences. The friction you hit. Why the obvious fix was not enough. Make the reader feel the constraint.]

**The solution**
[1–3 sentences. Name the system. State the creative approach in one clear idea.]

**How it works**
[3–6 sentences OR one short paragraph. Walk through the real implementation: inputs, process, guardrails, human gate. Minimize jargon.]

**The result**
[2–4 sentences. What is better now. What you can observe or measure. Stay honest if still early.]

**Bottom line**
[1–3 sentences. Portable lesson for another builder. Optional CTA.]
```

## Character budget (approximate)

| Section | Target share |
|---------|----------------|
| Header (title + project + link) | ~10% |
| The app | ~15% |
| The problem | ~15% |
| The solution | ~10% |
| How it works | ~30% |
| The result | ~12% |
| Bottom line | ~8% |

Hard cap: **2,500 characters** total.

## Naming pattern

Format: `[Product/Feature Name]: [plain benefit]`

Examples:

- Database Driven Social Content Engine
- UFIQ League Agent
- UFIQ Event Scheduler
- AI Picks Skill Optimization
