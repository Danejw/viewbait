# LinkedIn case study examples (ViewBait)

Use these as voice and structure references. Do not copy verbatim for new topics.

## Six-section format (canonical)

**File:** `docs/articles/viewbait-image-gen-masterclass/linkedin.md`

Structure:

1. Title + Project + Link
2. **The app** — ViewBait, thumbnail studio, what the core workflow does
3. **The problem** — prompt tools, wrong faces, style drift, unfair billing
4. **The solution** — Visual Brief Studio
5. **How it works** — form choices, brief behind the scenes, gallery, fair credits
6. **The result** — weekly consistency, one channel look
7. **Bottom line** — form over prompts, soft CTA

## Anti-patterns observed in drafts

| Avoid | Do instead |
|-------|------------|
| 8-step numbered philosophy list | One paragraph on the loop |
| "Learning beats reacting" triads | One concrete miss pattern |
| Stack dump in **The app** | User-facing feature description |
| "Not magic. Discovery." closer | Plain **Bottom line** lesson |
| Missing **Link:** under title | Always match case study header |
| Prose flow without section labels | Use all six bold section labels |
